Building a Lightweight Python SIEM with Discord Notifications

A cost-effective SIEM solution using Python and Discord webhooks for real-time security monitoring and alerting in Windows environments.

Introduction

In the modern cybersecurity landscape, continuous monitoring and rapid incident response are critical components of an organization's defense strategy. Security Information and Event Management (SIEM) systems serve as the central nervous system of security operations, providing the visibility and intelligence needed to detect, investigate, and respond to security threats.

This article explores the development of a lightweight, Python-based SIEM solution that leverages Discord webhooks for real-time alerting, making enterprise-grade security monitoring accessible to smaller organizations and individual security practitioners.

Understanding SIEM

What is SIEM?

Security Information and Event Management (SIEM) is a comprehensive security solution that aggregates, normalizes, and analyzes security event data from diverse sources across an organization's IT infrastructure. These sources include operating systems, applications, network devices, security appliances, and cloud services.

Core Functions of SIEM

  • Log Collection and Aggregation: Centralizes log data from servers, workstations, network devices, and security tools
  • Normalization and Parsing: Converts raw log data into a consistent format for effective correlation
  • Real-Time Monitoring: Analyzes incoming events against predefined rules and threat intelligence
  • Correlation and Analysis: Identifies patterns across multiple events and data sources
  • Compliance and Reporting: Maintains audit trails and generates comprehensive reports

Why Organizations Need SIEM

Without a SIEM solution, security teams face several challenges: visibility gaps where security events scattered across systems create blind spots, alert fatigue from thousands of daily alerts, slow incident response times, and compliance difficulties. Smaller organizations often lack the budget for enterprise solutions like Splunk, IBM QRadar, or Microsoft Sentinel, which can cost tens of thousands to millions of dollars annually.

Technical Architecture

System Architecture Flow

Windows Event Log
→
Event Monitor (pywin32)
→
Event Parser
→
Discord Webhook

The Python SIEM operates as a Windows service or standalone application that interfaces directly with the Windows Event Log subsystem. The architecture consists of three primary components:

  • Event Log Monitor: Uses the pywin32 library to access the Windows Event Log API, allowing real-time security event queries
  • Event Parser and Analyzer: Processes XML-formatted event data, extracting account names, timestamps, source systems, and metadata
  • Discord Notification Engine: Constructs JSON payloads conforming to Discord's webhook API and transmits via HTTPS

Monitored Security Events

The system focuses on seven critical Windows Event IDs that provide comprehensive visibility into authentication activities, account management, system changes, and potential data exfiltration vectors:

4624

Successful Logon

Captures all successful authentication attempts. Helps establish baseline user behavior and identify compromised credentials.

4625

Failed Logon

Indicates brute-force attacks, password guessing, or misconfigured applications. High volume followed by success suggests attack.

4720

User Account Created

Unauthorized account creation is a common persistence technique. Ensures immediate awareness of new accounts.

4723

Password Change Attempt

Records both successful and failed password changes. Helps detect unauthorized access and privilege escalation.

4724

Password Reset

Password resets by admin accounts can indicate account takeover. Critical for credential management audit trails.

11707

Application Installed

Unauthorized software represents security risk. Provides visibility into software changes on monitored systems.

6416

USB Device Inserted

USB devices present exfiltration risks and potential malware vectors. Helps prevent data loss and policy violations.

Implementation

Prerequisites

Requirements

  • Administrative Privileges: Required to access the Security Event Log
  • Windows Audit Policy: Must be configured to generate required security events
  • Discord Webhook: Create in the channel where alerts should appear

Installation

Deploy the Python SIEM by installing the required dependencies:

pip install pywin32 requests

The pywin32 package provides Windows API bindings for Event Log access, while requests handles HTTP communication with Discord's webhook API.

Configuring Windows Event Logging

Navigate to Local Group Policy Editor (gpedit.msc), then proceed to:

Computer Configuration
  → Windows Settings
    → Security Settings
      → Advanced Audit Policy Configuration

Enable these audit subcategories:

  • Account Logon: Enable "Audit Credential Validation" for Success and Failure
  • Account Management: Enable "Audit User Account Management" for Success
  • Logon/Logoff: Enable "Audit Logon" for Success and Failure
  • Object Access: Enable "Audit Removable Storage" for Success
  • Policy Change: Enable "Audit Audit Policy Change"

Force a Group Policy update:

gpupdate /force

Running the Application

python .\SIEM.py

Testing and Validation

Validate the deployment by generating test events:

net user john P@ssw0rd /add

This generates Event ID 4720, which should result in a Discord notification within seconds.

Security Considerations

Protecting the Webhook URL

The Discord webhook URL functions as an authentication token. If compromised, an attacker could spam false alerts. Protect it by:

  • Storing it in an encrypted configuration file or using Windows DPAPI
  • Restricting file system permissions on the configuration file
  • Rotating webhook URLs periodically and immediately if compromise is suspected

Rate Limiting and Alert Management

High-volume environments may generate thousands of events per hour. Implement these strategies:

  • Batch similar events occurring within short timeframes
  • Establish severity classifications to prioritize critical events
  • Configure aggregation rules for common event patterns

Monitoring the Monitor

SIEM systems themselves become critical infrastructure:

  • Verify the application process remains running
  • Test Discord webhook connectivity periodically
  • Generate heartbeat messages at regular intervals
  • Alert if the monitoring application fails

Extending the Solution

Adding Event Types

The modular architecture supports easy extension:

  • Identify new Event IDs using Event Viewer or Microsoft documentation
  • Add the Event ID to monitoring configuration
  • Implement parsing functions for the new event's XML structure
  • Design appropriate notification formats

Integration Options

  • SIEM Forwarding: Forward to enterprise SIEM systems like Splunk or Elastic
  • Ticketing Integration: Auto-create incidents in ServiceNow or Jira
  • Threat Intelligence: Enrich events with IP reputation and threat feeds

Conclusion

This lightweight Python SIEM demonstrates that effective security monitoring need not require enterprise-scale budgets or infrastructure. By focusing on high-value security events and leveraging readily available technologies like Discord for notifications, organizations can implement meaningful security monitoring with minimal investment.

While this implementation cannot replace enterprise SIEM systems in large, complex environments, it proves that the fundamental principles of security monitoring—continuous visibility, real-time alerting, and centralized analysis—can be achieved with accessible tools and straightforward implementations.
